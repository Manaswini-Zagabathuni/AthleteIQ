#  AthleteIQ — Smart Sports Performance & Analytics Database

> A professional-grade SQL system that tracks athletes, performance stats, training loads, injuries, and generates intelligent risk flags — simulating the backend of a real sports analytics platform.

---

##  Project Overview

**AthleteIQ** is an advanced SQL database project built for sports analytics. It goes beyond basic score tracking to model the data infrastructure that real teams and franchises use to manage athlete health, performance trends, and training optimization.

This project demonstrates:
- Complex relational schema design
- Advanced SQL (Window Functions, CTEs, Stored Procedures, Triggers)
- Real-world sports domain knowledge
- Clean project structure suitable for production systems

##  Key SQL Concepts Used

| Concept | Where Used |
|---|---|
| `WINDOW FUNCTIONS` (RANK, LAG, LEAD, AVG OVER) | top_performers.sql, peak_form_window.sql |
| `CTEs` (WITH clause) | All query files |
| `STORED PROCEDURES` + Cursors | calculate_player_rating.sql |
| `TRIGGERS` (AFTER INSERT) | injury_risk_alert.sql |
| `VIEWS` | injury_risk_alert.sql (vw_active_risk_flags) |
| `INDEXES` | All schema files |
| `ENUM`, `CHECK` constraints | schema/03_training.sql, schema/04_injuries.sql |
| Complex `JOINS` & Aggregations | team_comparison.sql, injury_impact_analysis.sql |
| `CASE WHEN` logic | Multiple query and trigger files |

##  Setup Instructions

### Prerequisites
- MySQL 8.0+ (recommended) or MariaDB 10.5+
- MySQL Workbench, DBeaver, or any SQL client

### Steps

1. **Clone the repository**
   ```bash
   git clone https://github.com/your-username/AthleteIQ.git
   cd AthleteIQ
   ```

2. **Run schema files in order**
   ```sql
   SOURCE schema/01_athletes.sql;
   SOURCE schema/02_performance.sql;
   SOURCE schema/03_training.sql;
   SOURCE schema/04_injuries.sql;
   ```

3. **Load seed data**
   ```sql
   SOURCE seed_data/sample_data.sql;
   ```

4. **Install procedures and triggers**
   ```sql
   SOURCE procedures/calculate_player_rating.sql;
   SOURCE triggers/injury_risk_alert.sql;
   ```

5. **Run queries**
   ```sql
   SOURCE queries/top_performers.sql;
   SOURCE queries/injury_impact_analysis.sql;
   SOURCE queries/peak_form_window.sql;
   SOURCE queries/team_comparison.sql;
   ```

##  Sample Outputs

### Top Performers Query
Returns athlete rankings with rolling averages and month-by-month rating trends using `RANK()` and `LAG()`.

### AthleteIQ Score (Stored Procedure)
```sql
CALL calculate_athlete_iq_score(1, @score, @grade, @summary);
SELECT @score, @grade, @summary;
```
Produces a composite 0–100 score graded S/A/B/C/D/F based on match ratings, recovery readiness, training load, goal contributions, and injury history.

### Active Risk Flags Dashboard (View)
```sql
SELECT * FROM vw_active_risk_flags;
```
Shows all unresolved injury/overtraining risk flags sorted by severity, auto-generated by triggers.
